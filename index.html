What is MKProjects

What's the best way to learn programming? It should be making projects and getting advice from it as you go on. What if you could create a project with support? That's the aim of MKProjects, and we hope to accomplish this in various ways. Whether it's creating books for the general public with the Creative Commons Attribution 4.0 International License making it opensource, or open source programs under the GPLv2.

Creativity should be endorsed to see it's full potential, there is no such thing as a bad idea, but an immature one

The Language Integration List

Here's a list of programming languages that are either present or planned to be integrated into future projects:

The usage of HTML is to build static, responsive web applications and it's mainly focused on the development of MKProj.com. MarkDown is mainly used to build the MKProject Books, and creates an easy development path to build web-books that are generated into HTML pages.

Knowledge is recommended for contributions

LaTEX is used to build attractive, professional documents and is found in MKProject Books, and will be used for any future manuals/documentations.

Knowledge is recommneded for contributions

Rust is the preferred language of development for developing applications such as Web, Command Line, etc. Rust also provides tools that are actively used, such as MDBook: which generates the web-books for MKProject-Books.

Knowledge is recommended for contributions

C++ is the 'beta' language sort of speak, which means something is first written in C++ before being reprodcued in Rust or Golang. However C++ is planned to take on a larger role for desktop application development and low-level applications development.

Dart & Flutter, it's cross-platform application framework is the perfect choice for development and will be the preferred choice for future mobile/desktop applications.

Python is currently used as a release packager, and is mainly being used as the middle man between the projects. Due to my preference of statically typed, compiled lanaguages. Python hasn't seen much of a role, however I expect it to be used more for future applications.
